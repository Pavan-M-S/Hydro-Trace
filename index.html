Overview

Abstract

HydroTrace is a dual-microcontroller IoT system that brings per-user accountability to public water dispensing stations. By combining RFID authentication, high-precision flow sensing, and wastewater hydrostatic measurement, the system captures every milliliter of dispensed and wasted water — linking it directly to individual user profiles in the cloud.

Why This Matters

At institutional scale, micro-wastage at water dispensing stations accumulates into thousands of liters of purified, energy-intensive water lost to drains each week. Traditional systems offer no visibility into who is wasting water, when, or how much.

  • No user accountability — water waste is anonymous and unlinked to individuals.
  • Passive infrastructure — conventional dispensers only detect major tank-level changes, not small but frequent overflows.
  • Behavioral blindness — without real-time feedback, users are unaware of their consumption patterns.
  • Data poverty — facility managers have no empirical data to drive conservation policies or ESG reporting.

Limitations of Current Approaches

Existing water monitoring solutions fall short in several critical dimensions — from measurement granularity to behavioral integration.

  • Coarse measurement — float switches and tank-level sensors have ±5–10% error margins, missing small but cumulative waste.
  • No user-level tracking — most systems measure aggregate flow, not per-user consumption, making it impossible to identify high-waste individuals.
  • No feedback loop — users receive no visual or behavioral feedback, so wasteful habits persist.
  • Infrastructure-heavy — many solutions require replacing existing dispensers, raising deployment costs and complexity.

Measurable Outcomes

HydroTrace is designed to deliver tangible environmental, financial, and behavioral impact across multiple dimensions.

  • Environmental — reduces purified water runoff by 70–80% in the first month of deployment, saving thousands of liters per station annually.
  • Financial — lowers water procurement and treatment costs, while reducing the energy footprint of pumping and filtration.
  • Behavioral — creates a psychological observer effect; when users know their consumption is tracked, they adjust behavior without mandates.
  • ESG & Compliance — provides empirical, user-linked water usage data for corporate sustainability reporting and green building certification (LEED, WELL).

Who Benefits

HydroTrace is built for any organization that operates public or semi-public water dispensing stations and wants to drive accountability and conservation.

  • Educational Institutions — universities, schools, and colleges with high-traffic water coolers and fountains.
  • Corporate Offices — ESG-conscious companies seeking to reduce operational water footprint and engage employees.
  • Public Facilities — airports, hospitals, government buildings, and community centers.
  • Commercial Real Estate — property managers aiming for green certifications and tenant satisfaction.
  • Hospitality Sector — hotels and resorts with multiple water dispensing points across their premises.

Hardware Architecture

Dual-microcontroller
sensor pipeline

Time-critical flow sensing stays on the Arduino; heavy Wi-Fi handshaking lives on the ESP32. No data lost, no pipeline collision.

🧠
Arduino Leonardo / Nano

Core local processor. Handles high-speed sensor interrupts — flow pulse counting, RFID reads — with no Wi-Fi latency interference.

📡
ESP32 Wi-Fi SoC

IoT network bridge. Handles secure REST API calls to Firebase, cloud authentication, and push notifications without stalling the sensor loop.

🔊
HC-SR04 Ultrasonic ×2

One monitors main clean water tank depth. The second watches the wastewater retention tank to prevent structural overflow events.

📐
Hydrostatic Level Sensor

Precision-calibrated inside the drain collector. Registers minor wastewater accumulation down to 1 mL displacement sensitivity.

🌀
YF-S201 Flow Sensor

Hall-effect sensor tracking live turbine rotations via pin interrupts. Calculates real-time volumetric output and flow rate during active dispensing.

🃏
RC522 RFID Module

Reads institutional ID card UIDs. Grants dispenser access only to registered profiles, logging every session to the user's cloud account.

🖥️
I2C 16×2 LCD

Shows the active user their dispensed vs wasted volume in real time, creating an immediate feedback loop that reinforces conservation behavior.

🔴
Multi-Stage LED Array

Color-coded status indicators for system state, user authentication success, wastage warning levels, and critical tank alerts.

🔥
Firebase Realtime DB

Cloud host for active user keys, aggregate data logs, and live telemetry flags. Drives both the admin web dashboard and push notification triggers.

System Design

Methodology & workflow

HydroTrace follows a systematic approach to capture, process, and visualize water usage data in real time.

1. User Authentication

RFID card is presented; the RC522 module reads the UID and sends it to the Arduino, which validates against the Firebase whitelist. Only authenticated users can start a dispensing session.

2. Flow Measurement

While the user dispenses water, the YF-S201 flow sensor generates pulses proportional to volume. The Arduino counts pulses via interrupts with microsecond precision.

3. Waste Capture

Any overflow or spilled water flows into the drain tray. The hydrostatic level sensor measures the resulting pressure change, calibrated to volume displacement (1 mL resolution).

4. Data Aggregation & Sync

The Arduino compiles dispensed volume, wasted volume, user ID, and timestamp. It sends the data packet to the ESP32, which pushes it to Firebase Realtime Database via REST API.

5. Real‑time Feedback

The LCD displays the user's current session metrics (dispensed vs wasted). The web dashboard polls Firebase every 500ms to update live charts, activity logs, and status indicators.

6. Alert & Action

If waste exceeds a configurable threshold or an anomaly (e.g., flow without RFID) is detected, the system triggers LED warnings and sends push notifications to facility managers.

Unique Differentiators

Beyond conventional
monitoring

HydroTrace introduces several breakthrough innovations that set it apart from any existing water accountability system.

🧪

Dual‑MCU Isolation

By separating real‑time sensor interrupts (Arduino) from Wi‑Fi communication (ESP32), we eliminate timing collisions and ensure zero data loss even under heavy network load.

📏

Hydrostatic + Ultrasonic Cross‑Calibration

Using two different sensing principles (pressure and time‑of‑flight) on the same physical parameter yields redundancy and self‑diagnosis, flagging drift or sensor failure automatically.

🔄

Offline‑First Logging

The Arduino buffers all session data in EEPROM. If Wi‑Fi drops, no data is lost — the ESP32 syncs the backlog upon reconnection with exponential backoff retries.

👤

User‑Level Accountability

Every drop is linked to a specific RFID identity, enabling granular behavior analysis, gamification, and targeted conservation campaigns — impossible with aggregate‑only systems.
